# 《人格心理学》教育智能体系统 — 功能分析与技术方案

## 文档说明

- **版本**：演示版（首期可落地迭代）
- **定位**：面向《人格心理学》课程的教学智能体应用，非通用聊天机器人；坚持立德树人、教学适切性与伦理合规。
- **安全提示**：本文档内含开发用 API Key。若仓库将公开或多人协作，请**立即将密钥移至环境变量**、从文档中删除明文，并在平台侧**轮换（重置）密钥**。

## 一、项目目标与教学问题映射

| 教学痛点 | 系统应对 |
|----------|----------|
| 概念会背、案例不会用 | 案例分析支架、多理论视角仿真、结构化作业反馈 |
| 课堂互动弱、个性化不足 | 任务链与学伴式问答、分层提示、讨论题/测验生成 |
| 过程性证据难收集 | 全链路学习日志、可导出字段、预留 NLP 编码接口 |
| 理论抽象、流派易混 | 理论对比模块、章节/流派过滤的知识库检索 |
| 评价薄弱 | 测验结果、案例分析记录、反思类文本的结构化存档 |

核心理念：**四课堂联动 + OBE/BOPPS 双层框架 + AI 助教 + 形成性评价闭环**（系统在首期先跑通数据采集与核心业务流）。

## 二、角色与权限

| 角色 | 能力概要 |
|------|----------|
| **学生** | 课程任务型功能（问答、对比、案例、角色模拟、测验）；不产生临床诊断与治疗建议 |
| **教师** | 知识库/案例库维护、学情与日志查看、合规话术与审核配置（首期可简化为策略配置 + 日志追踪） |
| **管理员**（可选） | 用户与角色分配、系统级安全策略（可与教师端合并为二期） |

## 三、功能清单（首期演示版）

### 3.1 学生端（≥5 项）

1. **课程问答（RAG）**  
   - 基于上传的大纲/教材/课件等做检索增强回答。  
   - 支持按**章节、理论流派、案例类型**过滤（首期可先实现章节 + 流派）。  
   - 系统提示词约束：教学辅导边界，拒绝诊断、贴标签、治疗处方。

2. **理论对比**  
   - 指定两个或多个流派/人物，输出维度化对比表（观点、方法、适用边界、局限）。  
   - 内容优先引用知识库；缺省时由模型生成但必须带“请先以课程 materials 为准”类提示。

3. **案例分析**  
   - 学生提交案例描述或使用教师预设案例。  
   - 输出：**分析框架（非诊断）**、可验证的观察点、理论联系、反思问题。  
   - 支持与「角色模拟」联动：同一案例切换多视角（精分、人本、特质、社会学习等）。

4. **角色模拟（虚拟仿真）**  
   - 基于模板的虚拟角色：**理论代表人物、来访者视角、咨询师互动对象、测评解释助手**（教学解释，非实测诊断）。  
   - 不少于一种多视角：同一简短案例脚本下切换理论立场进行对话或观点比较。

5. **章节测验**  
   - 客观题为主，题库来自教师维护；记录得分、用时、错题（可导出）。  
   - 可与知识库章节绑定。

**跨模块能力（渗透在以上功能中）**  
- **学科导师/学伴**：预习建议、概念讲解、难点辨析、复习路径（通过问答与对比体现）。  
- **教学辅助**：讨论题生成、测验题生成、案例分析框架、作业结构化反馈（可放在教师端「工具」或学生端「辅助任务」子区）。

### 3.2 教师端（≥4 项）

1. **知识库管理**  
   - 上传：大纲、教材节选、课件、讲义等。  
   - 切片、向量化、重建索引；标签：章节、流派、类型。  
   - 版本更新与增量更新（首期可先全量重建）。

2. **案例库管理**  
   - 案例增删改查、标签（教学用途/理论点）、脱敏说明字段。  
   - 可选用作学生案例分析题面或角色模拟脚本。

3. **学情统计**  
   - 按班级/时间：模块使用次数、会话轮次、测验均分、活跃时长等聚合指标。  
   - 预留导出为 CSV/JSON 供后续研究。

4. **日志查看**  
   - 检索：用户、模块、时间、会话 ID。  
   - 支持审计与内容安全复核（不作心理诊断用途的记录展示需备注教学场景）。

### 3.3 合规与安全（贯穿各端）

- **禁止**：心理/精神障碍**临床诊断**、治疗方案、药物建议、对个人的病理化标签。  
- **必须**：免责声明与教学用途说明；输出经规则 + 模型约束；敏感词与拒绝策略可配置。  
- **教师侧**：审核队列（二期可做）、配置项、操作与内容日志。

## 四、数据采集与形成性评价（首期字段预留）

用于后续 NLP 编码、能力画像、实验班对比研究，**首期至少记录**：

| 类别 | 建议字段 |
|------|----------|
| 身份与上下文 | `user_id`、角色、班级/课程班（可选） |
| 行为 | `module`（qa/compare/case/role/quiz…）、`session_id`、轮次、`duration_ms` |
| 内容类型 | `intent` 或 `task_type`（概念/对比/案例/测验…） |
| 业务结果 | 测验分数、案例提交 ID、作业反馈版本号 |
| 文本证据 | 提问摘要、案例分析文本、反思日志（脱敏策略可配置） |
| 技术 | 时间戳、客户端信息（可选）、RAG 命中文档 ID 列表（便于解释性） |

**导出**：教师端提供按时间范围/班级的 CSV/JSON 导出；API 预留 `GET /api/exports/learning-events`（权限校验）。

## 五、前后端分离与技术选型（建议）

### 5.1 总体架构

- **Web 演示版**：浏览器访问；**前端 SPA** + **REST/JSON API**（可选后续 WebSocket 用于流式对话）。  
- **RAG 管线**：上传 → 解析与分块 → Embedding → 向量库；问答时 hybrid（关键词 + 向量）检索 → 重排（可选）→ 带引用片段的生成。  
- **大模型**：国产兼容 OpenAI API 的接口（如 DeepSeek），统一 `LLMClient` 便于切换豆包/智谱等。

### 5.2 前端

| 项目 | 建议 |
|------|------|
| 框架 | **React 18 + TypeScript** |
| 构建 | **Vite** |
| UI | **Ant Design 5** 或 **antd + @ant-design/pro-components**（管理端表格友好） |
| 路由 | React Router |
| 状态 | Zustand 或 TanStack Query（服务端状态） |
| 富文本/对话 | 自研消息列表 + Markdown 渲染（如 react-markdown） |

学生端与教师端可用**同仓库多路由**（`/student/*`、`/teacher/*`）或子应用拆分（二期）。

### 5.3 后端

| 项目 | 建议 |
|------|------|
| 语言与框架 | **Python 3.11+**，**FastAPI**（异步、OpenAPI 文档、依赖注入） |
| 数据库 | **PostgreSQL**（用户、案例、测验、日志、元数据） |
| 向量检索 | **pgvector** 扩展 或 **独立 Milvus/Qdrant**（首期 pgvector 可降低运维） |
| 任务队列 | **Celery + Redis**（大文件解析、embedding 重建，可选首期同步简化） |
| 文件存储 | 本地目录或 **MinIO**（S3 兼容） |
| 鉴权 | JWT（Access + Refresh）或 Session；RBAC |

### 5.4 AI 与检索

- **Embedding**：如 bge-m3 / text-embedding-3 兼容接口（按国产部署或 API 可用性选择）。  
- **生成**：DeepSeek Chat（通过环境变量配置 Base URL 与 Key）。  
- **提示词**：按模块分模板（问答/对比/案例/角色/合规拒绝），统一插入课程边界与「非诊断」声明。

### 5.5 部署（演示）

- **Docker Compose**：`web`（nginx 托管静态）、`api`、`db`、`redis`。  
- 文档交付：`README.md`（环境变量、启动命令、默认账号说明）。
